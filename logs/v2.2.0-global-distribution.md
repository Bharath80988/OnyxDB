# v2.2.0 - Global Distribution & Packaging

## Overview
In version 2.2.0, we radically transformed OnyxDB from a source-code repository into a globally accessible, installable software package. The goal was to provide a seamless "one-command" setup experience for users across Java, Node.js, and Python ecosystems, while retaining a fully offline React dashboard.

## Changes

### 1. The Java Uber-Jar
- Integrated the `frontend-maven-plugin` into `onyxdb-api/pom.xml`.
- Maven now automatically downloads Node.js, installs dependencies, and compiles the React dashboard (`onyxdb-dashboard/dist`).
- The compiled React assets are copied into the Spring Boot `static/` directory during the `generate-resources` phase.
- **Result:** Executing `java -jar onyxdb.jar` starts the database API and serves the dashboard natively without requiring a separate Node.js server.

### 2. NPM Distribution Wrapper
- Created an `npm-wrapper` directory with an execution script (`index.js`).
- The script seamlessly checks the host for Java 21+ and automatically downloads the OnyxDB `.jar` file from GitHub Releases on its first run.
- Published to the NPM registry to allow frontend and fullstack developers to launch the database instantly via `npx onyxdb`.

### 3. Python PIP Distribution Wrapper
- Created a `pip-wrapper` directory containing standard Python packaging files (`setup.py`, `__init__.py`, `cli.py`).
- Implemented `urllib` / `requests` logic to replicate the NPM wrapper's behavior for Python environments.
- Allows AI and backend developers to start the database via `pip install onyxdb` and executing `onyxdb`.

### 4. Docker CI/CD Compatibility
- Resolved a conflict where Maven attempted to run the `frontend-maven-plugin` inside the `backend-build` Docker stage, causing architecture mismatches (`exit code 254`).
- Introduced a `<skip.frontend>` property in `pom.xml`, disabled by passing `-Dskip.frontend=true` in the `Dockerfile`.
- This ensures Docker continues to build the frontend optimally in Stage 1 using Alpine Node, while allowing local Maven builds to bundle the Uber-Jar seamlessly.
