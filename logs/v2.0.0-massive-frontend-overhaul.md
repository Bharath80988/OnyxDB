# v2.0.0 - Massive Frontend Overhaul

## Overview
Version 2.0.0 introduces a massive, commercial-grade frontend overhaul to the OnyxDB dashboard, documentation, and open-source transparency tracking. The goal was to provide a developer experience that looks stunning and feels incredibly responsive, inspired by industry leaders like MongoDB.

## Features Added

### 1. 10 Dynamic DaisyUI Themes
We transitioned from a simple light/dark toggle to a deeply integrated 10-theme system. The `<Navbar />` now contains a dynamic Theme Dropdown allowing users to switch themes on the fly.
- **Themes Available:** Light, Dark, Purple (Synthwave), Ocean (Aqua), Forest, Sunset, Rose (Valentine), Midnight (Night), Coffee, and Mint (Emerald).

### 2. Scrollable "Scrollspy" Documentation
The `/docs` route was completely reimagined. Instead of clicking through tabs, users experience a buttery smooth, single-page scrollable documentation site.
- Implemented **Lenis** for smooth cinematic scrolling.
- Added **Framer Motion** & **GSAP** for micro-animations as content enters the viewport.
- Added comprehensive connection tutorials and code snippets for **10 unique frameworks**: Java Spring Boot, Python Django, Flask, FastAPI, Node Express, PHP, Go Fiber, Ruby on Rails, C# ASP.NET Core, and Rust Actix.

### 3. Open Source Activity & Status Dashboard
A brand new `/status` route was created to highlight the project's open-source nature.
- **Activity Visualization:** Uses `Chart.js` to render a responsive line chart tracking mock commits and pull requests over the last 6 months.
- **Functionalities Grid:** A comprehensive roadmap matrix displaying what features are currently **Live** (e.g., B+ Tree Storage, Visual Builder) vs. what is **Planned** (e.g., Write-Ahead Logging, Vector Search, Raft Replication).

### 4. Global Aesthetic Enhancements
- Built a massive, multi-column global `<Footer />` component that persists across all application views.
- Updated the `README.md` to reflect the new capabilities and design philosophy.
- Extracted and tracked functionalities via `status/functionalities/implemented.md` and `status/functionalities/roadmap.md`.

## Security Patches
- Addressed GitGuardian warnings regarding hardcoded dummy Maven passwords within `settings.xml` by replacing them with environment variable placeholders (`${env.PROXY_PASS}` and `${env.REPO_PASS}`).

## Next Steps
With the visual layer fully modernized, the focus will shift back to backend robustness. Upcoming features will include Write-Ahead Logging (WAL) for durability and Vector Search for AI integrations.
