# v1.2.0 - B+ Tree Node Splitting

**Date:** 2026-07-18

## Changes
- **Feature:** `feat: Implement B+ Tree Node Splitting architecture`
- **File:** `BTreeManager.java`

## Details
- Introduced a 16-byte Page Header structure to identify node types (Leaf vs Internal) and track record counts.
- Implemented node splitting algorithm: When a Leaf Node hits 31 records (max capacity for an 8KB page at 256 bytes per record), it splits into two leaves and creates an internal root node.
- Implemented `nextLeafPageId` linked-list pointer for fast `scanAll` execution without climbing back up the tree.
- Tested successfully by inserting 35 records rapidly. The DB file `onyx.db` correctly allocated 3 pages (24KB).
